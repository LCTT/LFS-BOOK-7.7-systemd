i. 前言

本人学习并更好理解Linux的历程已有 10 多年，可以回退到 1998 年。当时，我刚刚安装了自己的首个 Linux 发行版，很快就对 Linux 所蕴含的整套概念和哲学理念产生了浓厚的兴趣。

完成一个单一的任务总是有很多的途径。对于 Linux 发行版亦是如此。多年来，已经存在了众多的发行版。一些仍然存在，一些已经变异成其它样子，有些则已经只存在于我们的记忆中。所有这些发行版均按不同的方 式行事，以满足其受众的需求。由于完成同样的最终目标具有多种不同的方式，我开始意识到自己可以不受限于任何别人的实现方式。在发现 Linux 之前，我们只能简单地忍受其它操作系统的问题，因为我们别无选择。事实就是如此，无论你喜不喜欢。在 Linux 下，开始出现选择的概念。如果您不喜欢什么，您可以自由地改变它，Linux甚至鼓励您这样做。

我尝试了多种发行版，无法确定使用任何一个。从其自身的角度来说，它们都是很伟大的系统。对和错不再是个问题。这已经成为一个个人的 习惯问题。在有着所有可能选择的情况下，显然没有哪个单一的系统于我而言最完美。于是我着手创建自己的 Linux 系统，以完全满足自己的喜好。

为了使其真正是我自己的系统，我毅然决定从源代码编译所有东西，而不是使用预编译好的二进制包。该“完美”的 Linux 系统将具不同系统的优点，而没有它们感知上的弱点。起初，这种想法相当让人畏惧，但我一直维持着可以构建出这样一个系统的信念。

在梳理了诸如循环依赖和编译错误之类的问题后，我最终构建出一个订制的 Linux 系统。它完全可以工作，并与当时已有的任何其它 Linux 系统一样完美可用。但它是我自己的创造。能够亲手组装出这样一个系统让人很满意。而唯一一个能让人感觉更美妙的事是能够亲自创建软件的每个部分。这是下一 件最美妙的事。

当我将自己的目标和经验与 Linux 社区的其它成员分享时，显然大家对这些想法的兴趣经久不衰。很快人们就清楚地意识到，这样一个定制的 Linux 系统不仅仅可以满足用户的特定需求，还可为程序员和系统管理员提供一个理想的学习机会，以增强他们（已有）的Linux技能。正是由于这种广泛的兴趣，Linux From Scratch Project 诞生了。

该 Linux From Scratch 手册是该项目的重中之重。它为您提供设计和构建自己的系统所需的背景知识和指令。本手册给出了一个样板，可以由此得到一个正确工作的系统，但您可自由改变 指令以适合自己的需求，从某种程度上说，这是本项目的一个重要组成部分。您仍旧可以控制；我们只是在您开启自己的旅程时助您一臂之力。

我真诚地希望您在自己的 Linux From Scratch 系统上能度过一段美好的时光，并能享受拥有一个真正属于自己的系统的众多好处。
