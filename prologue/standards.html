iv. LFS和标准

LFS的结构尽可能的和Linux的标准相同。 主要的标准是：

  • POSIX.1-2008.

  • 文件系统层次标准 版本3.0 草案1 (FHS)

  • Linux标准基础（LSB）规格

    LSB有五个独立的标准：内核，C++，桌面，运行时语言和输出。除了普通的要求，还有架构特定要求。LFS试图符合前一节中所讨论的架构。

    注意

    很多人不认同LSB的要求。定义它的主要目的是确保私有软件将能够在 兼容系统上安装并统正常运行。由于LFS是基于源代码的，用户对什么 软件包是需要的有完全的控制权，很多人选择不安装LSB指定的软件包。

创建一个能够通过LSB认证测试的完整LFS系统是可能的，但不能缺少 很多LFS范围之上的额外的软件包。在BLFS中有这些额外软件包的安装说明。

需要满足LSB要求的由LFS提供的软件包

LSB 内核:

Bash, Bc, Binutils, Coreutils, Diffutils, File, Findutils, Gawk, Grep, Gzip, M4, Man-DB, Ncurses, Procps, Psmisc, Sed, Shadow, Tar, Util-linux, Zlib

LSB C++:

Gcc

LSB 桌面:

无

LSB Runtime Languages:

Perl

LSB Printing:

无

LSB Multimeda:

无

需要满足LSB要求的由BLFS提供的软件包

LSB 内核:

At, Batch (a part of At), Cpio, Ed, Fcrontab, Initd-tools, Lsb_release, PAM, Pax, Sendmail (or Postfix or Exim), time

LSB C++:

无

LSB 桌面:

ATK, Cairo, Desktop-file-utils, Freetype, Fontconfig, Glib2, GTK+2, Icon-naming-utils, Libjpeg, Libpng, Libxml2, MesaLib, Pango, Qt4, Xorg

LSB运行时语言:

Python

LSB输出:

CUPS

LSB多媒体:

Alsa Libraries, NSPR, NSS, OpenSSL, Java, Xdg-utils

LSB 内核:

无

LSB C++:

无

LSB 桌面:

Qt3

LSB运行时语言:

无

LSB 输出:

无

LSB多媒体:

无
